Passenger Status.

Obtains the current status of Passenger by parsing results from passenger-status & passenger-memory-stats. The results are returned in the form of a hash.

This can be useful when load testing an application and you would like to record/log the results in a time series database amongst other uses.

*Only tested on Ubuntu Oneiric(11.10) so far but should work on other distros

Usage

gem install passenger_status

Example

require "rubygems"
require 'passenger_status'

status = PassengerStatus.passgener_stat
memory_status = PassengerStatus.passenger_memory

puts status.inspect
puts memory_status.inspect

Output Of Results

  1. Status(status.inspect)
{:queued=>"0", :sessions=>"0", :count=>"1",
 :processed=>"42", :active=>"0", :max=>"6", :inactive=>"1"}
  1. Memory Status(memory_status.inspect)

Parent Keys are PassengerSpawnServer **, **Rack, PassengerHelperAgent, PassengerWatchdog.

private & vm_size are used to obtain memory info for there respective parent keys.

{:PassengerSpawnServer=>{:private=>["9.4"], :vm_size=>["37.2"]}, 
:Rack=>{:private=>["92.7"], :vm_size=>["306.3"]}, 
:PassengerHelperAgent=>{:private=>["1.0"], :vm_size=>["100.7"]}, 
:PassengerWatchdog=>{:private=>["0.3"], :vm_size=>["87.2"]}}
